# Видение Luna

## Одной строкой

Luna — операционная система, которую хочется **использовать**, а не **настраивать**. Сначала это минимальный Linux-дистрибутив; в перспективе — shell и UI, построенные вокруг диалога с ИИ.

## Долгосрочная цель (финальный финал)

Интерфейс ОС как **чат с ИИ**: пользователь формулирует намерение («открой браузер», «найди файл с отчётом», «настрой Wi‑Fi»), система выполняет действие. Это не новое ядро — это **слой поверх userspace**: compositor, agent runtime, интеграция с файловой системой и сервисами.

```
┌─────────────────────────────────────┐
│  Luna Shell (AI-native UI)          │  ← далёкое будущее
├─────────────────────────────────────┤
│  Userspace: утилиты, сервисы, TUI   │
├─────────────────────────────────────┤
│  Init: OpenRC                       │
├─────────────────────────────────────┤
│  Kernel: Linux (Alpine)             │
└─────────────────────────────────────┘
```

Ядро Linux остаётся — мы кастомизируем **поведение и опыт**, а не переписываем paging и драйверы.

## Ближайшая цель (сейчас)

**Luna 0.4.0** — минимальный live-дистрибутив с узнаваемым брендингом и CLI `luna`:

1. ✅ Загружается с ISO в VirtualBox / QEMU
2. ✅ Рабочий терминал, сеть, `apk add`, `git clone`
3. ✅ Брендинг Luna (banner, MOTD, prompt)
4. ✅ CLI `luna` (version, status, help, tui)
5. ✅ Воспроизводимая сборка одной Docker-командой

**Следующий шаг:** [Фаза 4](roadmap.md#фаза-4--зачатки-ai-shell-далёкое-будущее) — AI-shell prototype. Стратегия UI без GNOME: [ui-strategy.md](ui-strategy.md).

## Принципы

1. **Минимализм** — только то, что нужно для текущего milestone; без «на всякий случай».
2. **Воспроизводимая сборка** — скрипт/Docker, а не ручные шаги в wiki.
3. **Итерации в VM** — каждое изменение проверяем загрузкой ISO; snapshot в VirtualBox.
4. **Отделение слоёв** — ядро/база (Alpine) → конфиг Luna → будущий Luna Shell.
5. **Учёба параллельно** — toy kernel / osdev можно изучать отдельно, не блокируя продукт.

## Чего Luna **не** будет на старте

- Собственного ядра с нуля
- Полноценного desktop environment (GNOME/KDE) — см. [ui-strategy.md](ui-strategy.md)
- Встроенного LLM и «умного» shell
- Поддержки всего железа (достаточно VirtualBox/QEMU)

## Что считаем успехом

| Этап | Критерий |
|------|----------|
| M0 | ISO загружается, login root/или user, `uname -a` показывает Luna |
| M1 | DHCP, ping, `apk`/`apt` работает внутри гостя |
| M2 | Свой motd, hostname, минимальный набор утилит |
| M3 | TUI-приложение или простой «Luna CLI» |
| M∞ | AI-native shell (отдельный большой проект) |
